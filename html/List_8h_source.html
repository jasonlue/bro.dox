4 // BaseList.h --
5 // Interface for class BaseList, current implementation is as an
6 // array of ent's. This implementation was chosen to optimize
7 // getting to the ent's rather than inserting and deleting.
8 // Also pairs of append's and get's act like push's and pop's
9 // and are very efficient. The only really expensive operations
10 // are inserting (but not appending), which requires pushing every
11 // element up, and resizing the list, which involves getting new space
12 // and moving the data. Resizing occurs automatically when inserting
13 // more elements than the list can currently hold. Automatic
14 // resizing is done one "chunk_size" of elements at a time and
15 // always increases the size of the list. Resizing to zero
16 // (or to less than the current value of num_entries)
17 // will decrease the size of the list to the current number of
18 // elements. Resize returns the new max_entries.
19 //
20 // Entries must be either a pointer to the data or nonzero data with
21 // sizeof(data) <= sizeof(void*).
