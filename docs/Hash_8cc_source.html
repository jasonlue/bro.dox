3 // The hash function works as follows:
4 //
5 // 1) For short data we have a number of universal hash functions:
6 // UHASH_CW (ax + b (mod p)), H3, Dietzfelbinger and UMAC_NH (UMAC_NH is
7 // not as strongly universal as the others, but probably enough). All
8 // these functions require number of random bits linear to the data
9 // length. And we use them for data no longer than UHASH_KEY_SIZE.
10 // They are faster than HMAC/MD5 used for longer data, and most hash
11 // operations are on short data.
12 //
13 // 2) As a fall-back, we use HMAC/MD5 (keyed MD5) for data of arbitrary
14 // length. MD5 is used as a scrambling scheme so that it is difficult
15 // for the adversary to construct conflicts, though I do not know if
16 // HMAC/MD5 is provably universal.
